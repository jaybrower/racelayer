# v0.1.3

_Released 2026-05-15_

## Added

- **Stint-scoped tire-deg trend in Pit Strategy** — Tire Deg section now tracks pace within the current stint (laps since last pit stop) rather than comparing against the session-best lap. Two numbers are shown:
  - **vs LAST N** (headline) — last lap compared to the rolling average of the prior up-to-3 clean stint laps. Positive = slower, negative = improving. Displayed in muted styling until a full 3-lap window is available.
  - **vs STINT BEST** (secondary) — direct delta between the last lap and the fastest lap of the current stint, so you always know how off-peak you are right now.
  - Section header shows **Stint of N** so you can see how deep into the current set you are at a glance.
- **Cockpit-only overlay rendering** — Overlays now hide themselves whenever you're in an iRacing menu (garage, get-in-car screen, replay viewer, spectator mode) and reappear automatically when you're back in your cockpit. Edit mode still shows everything so overlays can be positioned without being in a session.

## Internal

- `*.tsbuildinfo` added to `.gitignore` so TypeScript incremental-build artefacts don't clutter staged changes.
- New `Require Release Notes` GitHub Action enforces that every PR targeting a `release/v*` branch modifies the corresponding `release-notes/vX.Y.Z.md` file (bypass via the `no-release-notes` label). Keeps the changelog honest as features land.
- Historical release notes for v0.1.0 / v0.1.1 / v0.1.2 backfilled into `release-notes/` so the archive is complete from day one.
- New `Label Merged Issues` GitHub Action applies a `ready-to-release` label to issues referenced by feature PRs when those PRs merge into a `release/v*` branch. Lets the milestone view distinguish "not yet started" from "code merged, awaiting release" without closing issues prematurely.
