# v0.1.5

_Unreleased_

## Added

- **Settings: sidebar-tabbed layout.** The single long scrollable Settings page is now a sidebar nav (left) plus a scrollable pane (right), with six panes:
  - **General** ‚Äî Launch on startup, Auto-hide unsupported overlays, Overlay Positions reset.
  - **Overlays** ‚Äî per-session-type matrix grouped by overlay, with a sticky Practice / Qualifying / Race column header that never scrolls out of view, and a one-liner description above each overlay group.
  - **Shortcuts** ‚Äî Layout Mode and Open Settings keybindings (unchanged behaviour, dedicated pane).
  - **Preview Mode** ‚Äî enable + session type, with a description that explains what the feature is for.
  - **Updates** ‚Äî current version + update lifecycle (unchanged).
  - **About** ‚Äî version, links to the GitHub repo / changelog / new-issue form / MIT license.
  Closes #30.
- **Tooltips on overlay toggles.** Every row in the Overlays matrix has a native `title` tooltip with a one-line description of what the toggle does (e.g. *Side Indicator: amber chevrons next to your car number whenever iRacing flags a car alongside you*). Copy is centralised in `Settings/lib.ts` so future edits don't require chasing JSX.
- **Settings ‚Üí About ‚Äî external links open in the default browser** via a new `app:openExternal` IPC. Main-process side validates the URL is http(s) before calling `shell.openExternal`, so a compromised renderer can't fire arbitrary URL handlers.
- **Settings window remembers its size and position** across launches, keyed by monitor configuration ‚Äî same pattern overlays already use. Resize / move once, and that geometry comes back on every relaunch until you change it. Falls back to defaults on first launch and after a monitor-configuration change. Closes #37.
- **Settings: thin themed scrollbars** in place of the chunky OS-default scrollbar. The sidebar and content pane both use an 8px-wide scrollbar with a transparent track and a barely-visible thumb that brightens on hover ‚Äî fits the rest of the dark UI instead of fighting it.

## Changed

- **"Dev Mode" is now "Preview Mode"** throughout the app. The feature exists so users can preview and configure overlays without an active iRacing connection ‚Äî that's not development work, and the old name implied otherwise. User-visible surfaces updated: tray-menu checkbox ("Preview Mode") and session radios, Settings ‚Üí Preview Mode pane heading, status badge, toggle copy, and the in-Settings session-type label ("Preview Session Type"). No behaviour change. Part of #30.

## Fixed

- **Edit mode: overlays no longer grow while being dragged.** The custom drag handler used `BrowserWindow.setPosition()` only, which on Windows with DPI scaling let the OS reapply size constraints between frames and creep the window a pixel or two larger per frame. Width/height are now captured at drag-start and re-asserted on every move via `setBounds({x, y, width, height})`, so the size is pinned for the duration of the drag and the window ends at the exact dimensions it started. Closes #29.
- **Tray ‚Üî Settings sync for Preview Mode** (and other state-change broadcasts). The main-process `broadcastToAll` helper only iterated the overlay windows and silently skipped the Settings window, so toggling Preview Mode from the tray (or any future tray-driven state) never reached the Settings UI. The state would stay stale until the user closed and reopened Settings. Fan-out now includes the Settings window for state-change channels (`previewMode:changed`, `config:changed`, `update:status`), while high-frequency overlay-only events (`telemetry:update` at ~10 Hz, `overlay:editMode`) route through a separate `broadcastToOverlays` helper to avoid wasted IPC traffic. Pre-existing bug ‚Äî also affected the old `devMode:changed`.

## Internal

- **IPC rename**: `window:getPosition` / `window:setPosition` ‚Üí `window:getBounds` / `window:setBounds`, and matching preload helpers `getWindowBounds` / `setWindowBounds`. Bounds now include `width` and `height` so the renderer can lock size during a drag. No external API surface changed.
- **Preview Mode internal refactor** alongside the user-visible rename: `src/main/devMode.ts` ‚Üí `previewMode.ts`; `getDevMode` / `setDevMode` ‚Üí `getPreviewMode` / `setPreviewMode`; `DevModeState` ‚Üí `PreviewModeState`; IPC channels `devMode:get` / `devMode:set` / `devMode:changed` ‚Üí `previewMode:*`; preload helpers `getDevMode` / `setDevMode` / `onDevModeChanged` ‚Üí `getPreviewMode` / `setPreviewMode` / `onPreviewModeChanged`. No migration needed ‚Äî preview-mode state was already in-memory only and resets to off on every launch.
- **Settings file split**: the 683-line `Settings/index.tsx` is now a thin layout/state-holder; each pane lives under `Settings/panes/` (`GeneralPane.tsx`, `OverlaysPane.tsx`, `ShortcutsPane.tsx`, `PreviewModePane.tsx`, `UpdatesPane.tsx`, `AboutPane.tsx`), and pure helpers (`keyEventToAccelerator`, `formatAccelerator`, tooltip copy maps) moved to `Settings/lib.ts`. No behaviour change.
- **Manual test plan** (`docs/test-plan.md` ‚Üí Settings window) rewritten for the six-pane structure, with explicit checks for sticky-header behaviour, tooltip presence, About-pane external-link routing, and main-process scheme validation.
- **Manual test plan** (`docs/test-plan.md` ‚Üí Drag / layout mode) gained two explicit size-identity checks: full-monitor drag across 100% / 125% / 150% / 200% Windows scaling, and a cross-monitor drag on a mixed-scaling multi-monitor setup.
