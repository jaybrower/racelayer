## Fixed

- **Settings → Updates: "Check for updates" button no longer silently does nothing.** Previously, clicking the button could result in zero UI feedback in some paths — no `Checking…`, no error, no `Up to date` — because the renderer's state transitions were entirely event-driven from `autoUpdater`, and `autoUpdater` can silently resolve without firing any lifecycle events (notably in unpackaged dev runs, and via certain feed-fetch failures the library swallows). The button now (a) emits `Checking…` synchronously on click before calling `autoUpdater`, (b) has a 30 s safety-net timeout that surfaces an error if no follow-up event arrives, (c) detects the resolve-with-null silent path and surfaces a clear "Update server returned no version info" error, (d) shows the underlying error message under the row instead of just "Update failed", (e) renders a distinct "Running from source" state for unpackaged builds. Closes #46.

## Internal

- **`/release vX.Y.Z` slash command** at `.claude/commands/release.md` walks the full release pipeline end-to-end: finalize PR (release-notes `_Unreleased_` → `_Released_` + version-bump sanity), `release/* → main` merge, tag + push, `ready-to-release` milestone close-out, `npm run dist` build, GitHub Release creation with the `## Internal` section stripped from the user-facing body, artifact upload, and opening the next release branch + milestone. Documented inline with stop-and-ask gates around irreversible steps. Closes #44.
- **`npm run dist:pre` accepts a full version string**, not just a suffix. Detection by first character: digit → full semver-ish version used verbatim (e.g. `npm run dist:pre -- 0.1.0-autoUpdateTest.1`); letter → suffix appended to package.json version (e.g. `npm run dist:pre -- beta.1`, existing behaviour preserved). Surfaced while debugging #46 — needed a build *older* than the latest published release to test the update flow without temporarily editing `package.json`. Closes #48.
- **`electron-log` wired as `autoUpdater.logger`.** Full updater lifecycle now writes to `%APPDATA%/RaceLayer/logs/main.log` on installed builds, where there's no terminal to read console output from. Path also surfaced in the Updates pane footer as a support-debug affordance. Part of #46.
- **`dev-app-update.yml`** committed at repo root so a future debugging session can toggle off the `state: 'dev'` short-circuit in `src/main/updater.ts` and exercise the real GitHub feed in dev mode without setup. Part of #46.
