## Fixed

- **Settings Ôćĺ Updates: "Check for updates" button no longer silently does nothing.** Previously, clicking the button could result in zero UI feedback in some paths ÔÇö no `CheckingÔÇŽ`, no error, no `Up to date` ÔÇö because the renderer's state transitions were entirely event-driven from `autoUpdater`, and `autoUpdater` can silently resolve without firing any lifecycle events (notably in unpackaged dev runs, and via certain feed-fetch failures the library swallows). The button now (a) emits `CheckingÔÇŽ` synchronously on click before calling `autoUpdater`, (b) has a 30 s safety-net timeout that surfaces an error if no follow-up event arrives, (c) detects the resolve-with-null silent path and surfaces a clear "Update server returned no version info" error, (d) shows the underlying error message under the row instead of just "Update failed", (e) renders a distinct "Running from source" state for unpackaged builds. Additionally, the update channel is pinned to `latest` so `dist:pre` test builds (versions like `0.1.0-autoUpdateTest.1`) check the stable channel instead of deriving a non-existent custom channel from their prerelease tag. Closes #46.

## Internal

- **`/release vX.Y.Z` slash command** at `.claude/commands/release.md` walks the full release pipeline end-to-end: finalize PR (release-notes `_Unreleased_` Ôćĺ `_Released_` + version-bump sanity), `release/* Ôćĺ main` merge, tag + push, `ready-to-release` milestone close-out, `npm run dist` build, GitHub Release creation with the `## Internal` section stripped from the user-facing body, artifact upload, and opening the next release branch + milestone. Documented inline with stop-and-ask gates around irreversible steps. Closes #44.
- **`npm run dist:pre` accepts a full version string**, not just a suffix. Detection by first character: digit Ôćĺ full semver-ish version used verbatim (e.g. `npm run dist:pre -- 0.1.0-autoUpdateTest.1`); letter Ôćĺ suffix appended to package.json version (e.g. `npm run dist:pre -- beta.1`, existing behaviour preserved). Surfaced while debugging #46 ÔÇö needed a build *older* than the latest published release to test the update flow without temporarily editing `package.json`. Closes #48.
- **`electron-log` wired as `autoUpdater.logger`.** Full updater lifecycle now writes to `%APPDATA%/RaceLayer/logs/main.log` on installed builds, where there's no terminal to read console output from. File-level transport set to `debug` so electron-updater's own internal logs (HTTP URLs, response sizes, channel resolution, feed parsing) land in the same file. Path also surfaced in the Updates pane footer as a support-debug affordance. Part of #46.
- **Updater diagnostic probes.** On every `checkForUpdates` invocation, the updater module now logs a pre-flight state snapshot (`feedURL`, `channel`, `allowPrerelease`, `allowDowngrade`, `autoDownload`, current app version, packaged-status) before calling `autoUpdater.checkForUpdates()`. On rejection, a parallel raw-`https` probe of the Atom feed and `/releases/latest` URLs runs and logs the response status, redirect Location, content-type, and first 512 bytes of the body ÔÇö disambiguates "electron-updater HTTP-layer bug" from "host can't reach GitHub" / "GitHub returning unexpected data". Part of #46.
- **`dev-app-update.yml`** committed at repo root so a future debugging session can toggle off the `state: 'dev'` short-circuit in `src/main/updater.ts` and exercise the real GitHub feed in dev mode without setup. Part of #46.
