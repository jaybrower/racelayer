## Internal

- **`/release vX.Y.Z` slash command** at `.claude/commands/release.md` walks the full release pipeline end-to-end: finalize PR (release-notes `_Unreleased_` → `_Released_` + version-bump sanity), `release/* → main` merge, tag + push, `ready-to-release` milestone close-out, `npm run dist` build, GitHub Release creation with the `## Internal` section stripped from the user-facing body, artifact upload, and opening the next release branch + milestone. Documented inline with stop-and-ask gates around irreversible steps. Closes #44.
- **`npm run dist:pre` accepts a full version string**, not just a suffix. Detection by first character: digit → full semver-ish version used verbatim (e.g. `npm run dist:pre -- 0.1.0-autoUpdateTest.1`); letter → suffix appended to package.json version (e.g. `npm run dist:pre -- beta.1`, existing behaviour preserved). Surfaced while debugging #46 — needed a build *older* than the latest published release to test the update flow without temporarily editing `package.json`. Closes #48.
